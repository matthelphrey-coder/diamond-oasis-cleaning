
Replace with next/image using fill mode and preload:
```tsx
import Image from 'next/image'

export default function HeroSection() {
  return (
    <section className="relative min-h-[800px] lg:min-h-[700px] flex items-center">
      {/* Optimized hero image with preload for LCP */}
      <Image
        src="/images/hero-cleaning.jpg"
        alt="Professional house cleaning service in Las Vegas"
        fill
        priority={true}
        className="object-cover object-center"
        sizes="100vw"
        quality={85}
      />
      {/* Dark overlay for text readability */}
      <div className="absolute inset-0 bg-black/50" />

      {/* Content - same as before */}
      <div className="relative z-10 container mx-auto px-6 lg:px-12 py-12 lg:py-20">
        {/* ... rest of content unchanged ... */}
      </div>
    </section>
  )
}
```

Key changes:
1. Import Image from 'next/image'
2. Use Image component with fill={true} for full-section coverage
3. Add priority={true} for LCP preloading (Next.js 16 also accepts preload={true})
4. Add sizes="100vw" since image is full-width
5. Add quality={85} for good balance of quality/size
6. Add descriptive alt text with keywords
7. Remove inline style backgroundImage
8. Keep "use client" directive since InlineEstimateForm requires it

Note: Next.js 16.1.6 may use either priority or preload - check docs. If preload not recognized, use priority={true}.
